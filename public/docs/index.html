Introduction

Base URL: https://epk.anantatechnology.com

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

Endpoints

Register

Handle an incoming registration request.

Example request:
curl --request POST \
    "https://epk.anantatechnology.com/api/register" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"name\": \"jkar\",
    \"nidn_npm\": \"et\",
    \"email\": \"mhills@example.net\",
    \"telepon\": \"ryqypepqdbbb\",
    \"password\": \"ut\"
}"

const url = new URL(
    "https://epk.anantatechnology.com/api/register"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "name": "jkar",
    "nidn_npm": "et",
    "email": "mhills@example.net",
    "telepon": "ryqypepqdbbb",
    "password": "ut"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Request      

POST api/register

Headers

Content-Type      

Example: application/json

Accept      

Example: application/json

Body Parameters

name   string   

Must not be greater than 255 characters. Example: jkar

nidn_npm   string   

Example: et

email   string   

Must be a valid email address. Must not be greater than 255 characters. Example: mhills@example.net

telepon   string   

Must be at least 10 characters. Must not be greater than 13 characters. Example: ryqypepqdbbb

password   string   

Example: ut
